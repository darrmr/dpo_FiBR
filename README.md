# Практическое занятие 2: Верстка сайта по макету на HTML

---

## Цель
Освоить принципы семантической HTML-разметки, научиться создавать корректную структуру веб-страницы на основе готового макета. Получить навыки подключения внешних ресурсов, работы с типографикой и валидации HTML-кода. Закрепить на практике работу с Git и публикацию проекта на GitHub Pages.

---

## Введение
Современная фронтенд-разработка требует не просто создания визуально привлекательных интерфейсов, но и построения доступной, понятной для поисковых систем и легко поддерживаемой структуры. Основа такого подхода — семантическая вёрстка.

## Ключевые концепции и теги для вёрстки

### Секционные теги
Секционные теги делят HTML-документ (страницу) на большие логические блоки или секции:

- `<header>` — задаёт «шапку» сайта или раздела. Содержит вводную информацию: логотип, навигацию, поиск.
- `<nav>` — определяет блок основной навигации по сайту (меню).
- `<main>` — заключает в себе основное, уникальное содержимое страницы. На странице должен быть только один `<main>`.
- `<section>` — определяет тематический раздел документа. Обычно имеет собственный заголовок.
- `<article>` — предназначен для самостоятельного, независимого фрагмента контента: новость, пост в блоге, карточка товара.
- `<aside>` — содержит контент, косвенно связанный с основным (сайдбары, боковые панели, блоки с рекламой).
- `<footer>` — задаёт «подвал» сайта или раздела. Часто содержит контактную информацию, ссылки на соцсети, копирайт.

### Теги для работы с контентом
- **Иерархия заголовков (`<h1>`-`<h6>`)** — `<h1>` должен быть один на странице и обозначать её главную тему. Далее заголовки идут по убыванию важности, формируя четкую структуру документа.
- `<p>` — параграф, блок текста.
- `<ul>`, `<ol>`, `<li>` — маркированные и нумерованные списки.
- `<a>` — ссылка. Обязательный атрибут `href` указывает URL-адрес.
- `<img>` — изображение. Обязательные атрибуты: `src` (путь к файлу) и `alt` (альтернативный текст).
- `<button>` — кнопка для взаимодействия пользователя.

### Работа с ресурсами и метаинформацией
- **Подключение шрифтов** — современный стандарт — использование правила `@font-face` в CSS для подключения кастомных шрифтов формата WOFF2. Будет затронуто в Практическом занятии 3.
- **Мета-тег viewport** — `<meta name="viewport" content="width=device-width, initial-scale=1.0">` важен для корректного отображения и работы сайта на мобильных устройствах.
- **Фавиконка** — `<link rel="icon" type="image/x-icon" href="/assets/icons/favicon.ico">` — маленькая иконка, отображаемая во вкладке браузера.

### Валидация кода
Это процесс проверки HTML- и CSS-кода на соответствие официальным стандартам консорциума W3C. Валидный код гарантирует его корректное отображение в разных браузерах и является признаком качественной разработки.

Подробнее можно ознакомится в файле презентации (Лекция 2).

## Последовательность выполнения работы

### Шаг 1. Подготовка репозитория и структуры проекта
1. Создайте новую папку для проекта или используйте репозиторий с предыдущей практики.
2. Если создаёте новый репозиторий, выполните инициализацию Git, свяжите с удалённым репозиторием на GitHub и настройте GitHub Pages (см. Практическое занятие 1).
3. Создайте внутри репозитория следующую структуру папок и файлов (в рамках выполнения Практического занятия 3 она будет дополнена).

### Шаг 2. Выбор и анализ макета
Вам предоставлены два варианта макета. Выберите один из них для вёрстки. Внимательно изучите выбранный макет (представлен в .pdf в соответствующем архиве) и мысленно или на бумаге разбейте его на семантические блоки:

- Где находится `<header>`? Что в него входит (лого, навигация)?
- Что является `<main>` содержанием?
- Сколько на странице `<section>`? Как они логически разделены?
- Есть ли на макете `<aside>`?
- Где находится `<footer>`? Что он содержит?

Обратите внимание на то, что в архиве приложены также файлы с дизайн-системой проекта. Они понадобятся для реализации практики 3.

### Шаг 3. Базовая HTML-разметка
1. В файле `index.html` создайте базовый каркас HTML5 документа (`! + Tab` в VS Code).
2. Убедитесь, что в `<head>` прописаны:
   - `<meta charset="UTF-8">`
   - `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
   - `<title>` с названием вашего сайта
3. Начните наполнять `<body>` семантическими тегами в соответствии с проанализированной структурой. Пока не обращайте внимания на содержимое, просто «набросайте» каркас.

```html
 <header>...</header>
 <main> 
<section>...</section> 
<section>...</section>
 </main>
 <footer>...</footer>
 ```

4. Наполните каждый блок соответствующим контентом: заголовками, параграфами, изображениями, списками. Используйте предоставленные материалы.
5. В примере представлен иностранный текст. Вы можете заменить его сгенерированной текстурой (используйте русскоязычную генерацию, например, на РыбаТекст).

### Шаг 4. Работа с типографом + валидация
1. **Типограф** — для корректного отображения текста (кавычки, тире, дефисы) обработайте все текстовые блоки через сервис-типограф, например, `articleedev.ru/typograf/` или `typograf.ru`. Это не позволит разрывать названия и имена на две строчки, заменит прямые кавычки "..." на «ёлочки» для русского текста, поставит правильные тире и неразрывные пробелы.
2. **Валидация** — после завершения базовой вёрстки проверьте ваш HTML-код с помощью Validator W3C. Исправьте все ошибки и предупреждения.

### Шаг 5. Публикация и отчётность
1. Зафиксируйте все изменения в Git.
2. Убедитесь, что GitHub Pages обновился и ваша работа корректно отображается по публичной ссылке.
3. В файл `README.md` добавьте описание проекта, используемые технологии (пока что только HTML5) и активную ссылку на опубликованную страницу.

## Дополнительное задание (для углубленного изучения)
**Оптимизация изображений** — с помощью инструментов (например, `Squoosh.app`) оптимизируйте все изображения в макете, не теряя видимого качества. Это ускорит загрузку страницы.

## Критерии оценивания

- **Структура проекта** — корректная файловая структура проекта, наличие всех необходимых папок (`assets/img`, `assets/icons` и др.)
- **Семантическая вёрстка** — использование тегов `<header>`, `<main>`, `<footer>`, `<section>`, `<nav>`, `<article>` по назначению. Наличие только одного `<h1>` на странице
- **Подключение ресурсов** — корректное отображение всех изображений из макета
- **Валидность кода** — HTML-код проходит проверку на валидаторе W3C без критических ошибок
- **Типографика** — текст обработан типографом (правильные кавычки, тире)
- **Работоспособность** — проект опубликован на GitHub Pages, ссылка в `README.md` ведёт на рабочую страницу
